openrvdas.server.logger_manager

Manage the loggers defined by an OpenRVDAS cruise definition.

LoggerManager is the central process-control component of OpenRVDAS. It reads desired logger configurations from a ServerAPI-based data store (in-memory, SQLite or Django), runs each logger configuration in its own process via a LoggerRunner, restarts loggers that die unexpectedly, and reports logger status to a CachedDataServer.

May be invoked from the command line as, e.g.

    server/logger_manager.py         --config test/NBP1406/NBP1406_cruise.yaml         --mode monitor         --start_data_server

which will load the specified cruise definition, set it to the specified mode and start a command line console for issuing commands. Type "help" at the console for a list of commands.

In production it is typically run via supervisord with a persistent database backing it:

    server/logger_manager.py --database django --no-console         --data_server_websocket :8766

 97    ############################
 98    def __init__(self,
 99                 api, data_server_websocket=None,
100                 stderr_file_pattern='/var/log/openrvdas/{logger}.stderr',
101                 max_tries=DEFAULT_MAX_TRIES, min_uptime=DEFAULT_MIN_UPTIME,
102                 interval=0.5, logger_log_level=logging.WARNING):
103        """Read desired logger configs from the data store and try to keep
104        the loggers specified in those configs running.
105        ```
106        api - ServerAPI (or subclass) instance by which LoggerManager will get
107              its data store updates
108
109        data_server_websocket - cached data server host:port to which we are
110              going to send our status updates.
111
112        stderr_file_pattern - Pattern into which logger name will be
113              interpolated to create the file path/name to which the
114              logger's stderr will be written. E.g.
115              '/var/log/openrvdas/{logger}.stderr' If
116              data_server_websocket is defined, will also write logger
117              stderr to it.
118
119        max_tries - number of times to try restarting a dead logger config
120              before giving up on it as failed. If zero, never stop retrying.
121
122        min_uptime - how many seconds a logger must run to count as having
123              been successfully started and reset its restart count.
124
125        interval - number of seconds to sleep between checking/updating
126              loggers
127
128        logger_log_level - at what logging level our component loggers
129              should operate.
130        ```
131        """
132        # Set signal to catch SIGTERM and convert it into a
133        # KeyboardInterrupt so we can shut things down gracefully.
134        try:
135            signal.signal(signal.SIGTERM, kill_handler)
136        except ValueError:
137            logging.warning('LoggerManager not running in main thread; '
138                            'shutting down with Ctl-C may not work.')
139
140        # api class must be subclass of ServerAPI
141        if not issubclass(type(api), ServerAPI):
142            raise ValueError('Passed api "%s" must be subclass of ServerAPI' % api)
143        self.api = api
144
145        # Data server to which we're going to send status updates and to
146        # which our loggers should send their stderr.
147        self.data_server_websocket = data_server_websocket
148        if data_server_websocket:
149            self.data_server_writer = CachedDataWriter(data_server_websocket)
150
151            # Also send our own stderr to the data server so the console
152            # can display it.
153            cds_writer = ComposedWriter(
154                transforms=ToDASRecordTransform(data_id='stderr',
155                                                field_name='stderr:logger_manager'),
156                writers=self.data_server_writer)
157            logging.getLogger().addHandler(StdErrLoggingHandler(cds_writer))
158        else:
159            self.data_server_writer = None
160
161        self.stderr_file_pattern = stderr_file_pattern
162        self.max_tries = max_tries
163        self.min_uptime = min_uptime
164        self.interval = interval
165        self.logger_log_level = logger_log_level
166
167        # How our various loops and threads will know it's time to quit
168        self.quit_flag = False
169
170        # Where we store the latest cruise definition and status reports.
171        self.cruise = None
172        self.cruise_filename = None
173        self.cruise_loaded_time = 0
174
175        self.loggers = {}
176
177        self.logger_status = None
178        self.status_time = 0

181        # logger should be running, the LoggerRunner running it and its
182        # restart bookkeeping. Guarded by config_lock.
183        self.logger_states = {}

194    def start(self):
195        """Start the threads that make up the LoggerManager operation:
196
197        1. Loop to update configs from the API, check that the loggers
198           are running what they should be (restarting any that have died
199           unexpectedly), and send status reports to the cached data server.
200        2. Loop to check whether the cruise definition has changed and, if
201           so, send the updated definition to the cached data server.
202
203        Start threads as daemons so that they'll automatically terminate
204        if the main thread does.
205        """
206        logging.info('Starting LoggerManager')
207
208        self.run_loggers_thread = threading.Thread(
209            name='run_loggers_loop',
210            target=self._run_loggers_loop, daemon=True)
211        self.run_loggers_thread.start()
212
213        self.send_cruise_definition_thread = threading.Thread(
214            name='send_cruise_definition_loop',
215            target=self._send_cruise_definition_loop, daemon=True)
216        self.send_cruise_definition_thread.start()

219    def quit(self):
220        """Exit the loops and shut down all loggers."""
221        self.quit_flag = True
222        with self.config_lock:
223            for logger in list(self.logger_states):
224                self._delete_logger(logger)
225
226        # Give our data server writer a chance to deliver anything still
227        # queued (e.g. the loggers' final stderr lines) before we exit.
228        if self.data_server_writer:
229            self.data_server_writer.close()

232    def update_configs(self):
233        """Fetch the latest desired configs from the API and start/stop
234        loggers as necessary to match them.
235
236        Called periodically from _run_loggers_loop(), and also registered
237        as a callback so the API can invoke it when it knows the active
238        configs have changed. Search for the line:
239
240          api.on_update(callback=logger_manager.update_configs)
241
242        in this file to see where.
243        """
244        # Get new configs in dict {logger: config_spec}
245        logger_configs = self.api.get_logger_configs()
246        if logger_configs:
247            self._apply_configs(logger_configs)

249    ############################
250    def _apply_configs(self, configs):
251        """Receive a new map of {logger: config} and start/stop loggers as
252        necessary to match it.
253        """
254        with self.config_lock:
255            # If we're in the process of quitting, go home - a different
256            # thread is already shutting things down.
257            if self.quit_flag:
258                return
259
260            stale_loggers = set(self.logger_states) - set(configs)
261            new_loggers = set(configs) - set(self.logger_states)
262            other_loggers = set(self.logger_states) - stale_loggers - new_loggers
263
264            logging.debug('Stale: %s', stale_loggers)
265            logging.debug('New: %s', new_loggers)
266            logging.debug('Other: %s', other_loggers)
267
268            # Find and shut down loggers that don't exist in our new configs
269            for logger in stale_loggers:
270                logging.info('Shutting down logger %s.', logger)
271                self._delete_logger(logger)
272
273            # Add loggers that have first appeared in our new config and
274            # start them up.
275            for logger in new_loggers:
276                new_config = configs[logger]
277                logging.info('Starting new logger %s with %s.', logger,
278                             new_config.get('name', 'no_name'))
279                self._start_logger(logger, new_config)
280
281            # For existing loggers, see whether their configs have
282            # changed. If so, stop and restart with new config.
283            for logger in other_loggers:
284                new_config = configs[logger]
285                old_config = self.logger_states[logger].config
286                if new_config == old_config:
287                    logging.debug('Config for %s unchanged.', logger)
288                    continue
289
290                logging.info('Updating %s from %s to %s', logger,
291                             old_config.get('name', 'no_name'),
292                             new_config.get('name', 'no_name'))
293                self._delete_logger(logger)
294                self._start_logger(logger, new_config)
295
296            self.active_configs = configs

299    def _start_logger(self, logger, config):
300        """Create and start a LoggerRunner for the passed config. ONLY CALL
301        THIS WITH config_lock HELD, for thread safety."""
302        config_name = config.get('name', logger + '_config')
303        logging.info('Called start_logger for %s: %s', logger, config_name)
304
305        stderr_filename = self.stderr_file_pattern.format(logger=logger)
306
307        # The runner captures the logger's actual stderr (fd 2) via a pipe
308        # and relays each line back to us; we forward them to the cached
309        # data server over our own connection, so the (possibly dying)
310        # logger process is never responsible for delivering its own
311        # death message.
312        def stderr_callback(line, logger=logger):
313            self._forward_logger_stderr(logger, line)
314
315        # The relay thread also tells us - via pipe EOF - when the process
316        # has died, so we can restart it immediately instead of waiting
317        # for the next polling check.
318        def death_callback(runner, logger=logger):
319            self._handle_logger_death(runner, logger)
320
321        runner = LoggerRunner(config=config, name=logger,
322                              stderr_filename=stderr_filename,
323                              stderr_callback=stderr_callback,
324                              death_callback=death_callback,
325                              logger_log_level=self.logger_log_level)
326        self.logger_states[logger] = LoggerState(config=config, runner=runner)
327        runner.start()

330    def _delete_logger(self, logger):
331        """Shut down the named logger and forget about it. ONLY CALL THIS
332        WITH config_lock HELD, for thread safety."""
333        state = self.logger_states.get(logger)
334        if not state:
335            logging.warning('Stale logger %s not found?!?', logger)
336            return
337        logging.info('Waiting for logger %s to complete', logger)
338        state.runner.quit()
339        del self.logger_states[logger]

341    ############################
342    def _check_loggers(self):
343        """Polling safety net: check that all the loggers we ought to be
344        running are running, and restart any that have died unexpectedly.
345        The primary death signal is _handle_logger_death(), called by each
346        runner's stderr relay thread when its pipe hits EOF; this loop
347        catches anything that slips past it.
348        """
349        logging.debug('Checking loggers...')
350        with self.config_lock:
351            for logger, state in self.logger_states.items():
352                self._consider_restart(logger, state)
353
354    ############################
355    def _consider_restart(self, logger, state):
356        """If the passed logger is unexpectedly dead, restart it - unless it
357        has died max_tries times without staying up at least min_uptime
358        seconds, in which case declare it failed and stop trying. ONLY CALL
359        THIS WITH config_lock HELD, for thread safety.
360        """
361        runner = state.runner
362        if not runner.is_runnable():
363            return
364        if runner.is_alive():
365            return
366        if runner.is_failed():
367            return
368
369        # If we're here, runner is runnable, is not running, and hasn't
370        # yet been labeled as a failed logger.
371        logging.warning('%s unexpectedly dead.', logger)
372
373        # How long was it up? If long enough, give it a clean slate.
374        if time.time() - state.last_started < self.min_uptime:
375            state.restart_count += 1
376        else:
377            state.restart_count = 0
378
379        # If we've restarted too many times recently, declare the logger
380        # failed and move on. max_tries of zero means never stop retrying.
381        if self.max_tries and state.restart_count >= self.max_tries:
382            runner.failed = True
383            logging.warning('%s has failed %s times; not restarting',
384                            logger, state.restart_count)
385            return
386
387        # If here, we're going to try restarting.
388        logging.info('%s - restarting', logger)
389        state.last_started = time.time()
390        runner.start()
391
392    ############################
393    def _handle_logger_death(self, runner, logger):
394        """Called - from the dead runner's stderr relay thread - when a
395        logger process's stderr pipe hits EOF, i.e. the process has exited
396        and its last words have been delivered. Restart it (or declare it
397        failed) right away rather than waiting for the next polling check.
398        """
399        if self.quit_flag or runner.quit_flag:
400            return  # deliberate shutdown, not a death
401
402        # Acquire the lock with a timeout: a manager thread holding it may
403        # be in runner.quit(), joining the very relay thread we're running
404        # in. Re-checking the quit flags lets us bail out instead of
405        # stalling that join.
406        while not self.config_lock.acquire(timeout=0.5):
407            if self.quit_flag or runner.quit_flag:
408                return
409        try:
410            state = self.logger_states.get(logger)
411            if not state or state.runner is not runner:
412                return  # stale notification; logger was reconfigured
413            self._consider_restart(logger, state)
414        finally:
415            self.config_lock.release()

418    def get_status(self):
419        """Return a dict of the current config name and current run status of
420        each logger in the form, e.g.:
421
422        {'s330': {'config':'s330->net', 'status':'RUNNING'},
423         'gyr1': {'config':'gyr1->file', 'status':'FAILED'},
424        }
425
426        Possible status are EXITED, RUNNING, FAILED and STARTING. EXITED
427        is the status when a logger is not 'runnable' - e.g. the 'off'
428        config. STARTING is used when a runner is runnable but is not
429        running and is not FAILED - i.e. we haven't given up on it.
430        """
431        logger_status = {}
432        with self.config_lock:
433            for logger, state in self.logger_states.items():
434                config_name = state.config.get('name', 'no name')
435
436                if not state.runner.is_runnable():
437                    status = 'EXITED'
438                elif state.runner.is_alive():
439                    status = 'RUNNING'
440                elif state.runner.is_failed():
441                    status = 'FAILED'
442                else:
443                    status = 'STARTING'
444                logger_status[logger] = {'config': config_name, 'status': status}
445        return logger_status

448    def _run_loggers_loop(self):
449        """Main supervision loop: update configs from the API, restart any
450        loggers that have died unexpectedly, and send status reports to the
451        cached data server.
452        """
453        while not self.quit_flag:
454            try:
455                self.update_configs()
456                self._check_loggers()
457                self._send_status()
458            except (AttributeError, KeyError, TypeError, ValueError) as e:
459                logging.warning('Error in logger update loop: %s', e)
460            time.sleep(self.interval)

477    def load_definition_from_api(self):
478        """Fetch a new cruise definition from the API and build local maps.
479        Then send an updated cruise definition to the console.
480
481        Called from _send_cruise_definition_loop() when it notices a new
482        definition has been loaded, and also registered as a callback so
483        the API can invoke it when a new definition is loaded. Search for
484        the line:
485
486          api.on_load(callback=logger_manager.load_definition_from_api)
487
488        in this file to see where.
489        """
490        logging.info('Fetching new cruise definitions from API')
491        try:
492            self.loggers = self.api.get_loggers()
493
494            self.cruise = self.api.get_configuration()  # a Cruise object
495            self.cruise_filename = self.cruise.get('config_filename')
496            loaded_time = self.cruise.get('loaded_time')
497            self.cruise_loaded_time = datetime.datetime.timestamp(loaded_time)
498            self.active_mode = self.api.get_active_mode()
499
500            # Send updated cruise definition to CDS for console to read.
501            cruise_dict = {
502                'cruise_id': self.cruise.get('id', ''),
503                'filename': self.cruise_filename,
504                'config_timestamp': self.cruise_loaded_time,
505                'loggers': self.loggers,
506                'modes': self.cruise.get('modes', {}),
507                'active_mode': self.active_mode,
508            }
509            logging.info('Sending updated cruise definitions to CDS.')
510            self._write_record_to_data_server(
511                'status:cruise_definition', cruise_dict)
512        except (AttributeError, ValueError, TypeError) as e:
513            logging.info('Failed to update cruise definition: %s', e)

516    def _send_cruise_definition_loop(self):
517        """Iteratively check whether the cruise definition in the data store
518        (or its source file) has changed. If the file has changed, send a
519        notification to the console so it can ask whether the user wants to
520        reload. If the data store has a definition with a newer timestamp,
521        rebuild our local maps and send the updated definition to the
522        cached data server.
523        """
524        last_loaded_timestamp = 0
525
526        while not self.quit_flag:
527            try:
528                self.cruise = self.api.get_configuration()  # a Cruise object
529                if not self.cruise:
530                    logging.info('No cruise definition found in API')
531                    time.sleep(self.interval * 2)
532                    continue
533                self.cruise_filename = self.cruise.get('config_filename')
534                loaded_time = self.cruise.get('loaded_time')
535                self.cruise_loaded_time = datetime.datetime.timestamp(loaded_time)
536
537                # Has cruise definition file changed since we loaded it? If so,
538                # send a notification to console so it can ask if user wants to
539                # reload.
540                if self.cruise_filename:
541                    try:
542                        mtime = os.path.getmtime(self.cruise_filename)
543                        if mtime > self.cruise_loaded_time:
544                            logging.debug('Cruise file timestamp changed!')
545                            self._write_record_to_data_server('status:file_update', mtime)
546                    except FileNotFoundError:
547                        logging.debug('Cruise file "%s" has disappeared?', self.cruise_filename)
548
549                # Does database have a cruise definition with a newer timestamp?
550                # Means user loaded/reloaded definition. Update our maps to
551                # reflect the new values and send an updated cruise_definition
552                # to the console.
553                if self.cruise_loaded_time > last_loaded_timestamp:
554                    last_loaded_timestamp = self.cruise_loaded_time
555                    logging.info('New cruise definition detected - rebuilding maps.')
556                    self.load_definition_from_api()
557
558            except (AttributeError, ValueError, TypeError) as e:
559                logging.warning('Error checking cruise definition: %s', e)
560
561            # Whether or not we've sent an update, sleep
562            time.sleep(self.interval * 2)

565    def _forward_logger_stderr(self, logger, line):
566        """Receive one line of a logger's stderr from its LoggerRunner's
567        relay thread and forward it to the cached data server. Lines arrive
568        already formatted/timestamped by the logger process itself.
569        """

571            record = DASRecord(data_id='stderr',
572                               fields={'stderr:logger:' + logger: line})
573            self.data_server_writer.write(record)

583        else:
584            logging.info('Update: %s: %s', field_name, record)

589def run_data_server(data_server_websocket,
590                    data_server_back_seconds, data_server_cleanup_interval,
591                    data_server_interval):
592    """Run a CachedDataServer (to be called as a separate process),
593    accepting websocket connections to receive data to be cached and
594    served.
595    """

356    def _consider_restart(self, logger, state):
357        """If the passed logger is unexpectedly dead, restart it - unless it
358        has died max_tries times without staying up at least min_uptime
359        seconds, in which case declare it failed and stop trying. ONLY CALL
360        THIS WITH config_lock HELD, for thread safety.
361        """
362        runner = state.runner
363        if not runner.is_runnable():
364            return
365        if runner.is_alive():
366            return
367        if runner.is_failed():
368            return
369
370        # If we're here, runner is runnable, is not running, and hasn't
371        # yet been labeled as a failed logger.
372        logging.warning('%s unexpectedly dead.', logger)
373
374        # How long was it up? If long enough, give it a clean slate.
375        if time.time() - state.last_started < self.min_uptime:
376            state.restart_count += 1
377        else:
378            state.restart_count = 0
379
380        # If we've restarted too many times recently, declare the logger
381        # failed and move on. max_tries of zero means never stop retrying.
382        if self.max_tries and state.restart_count >= self.max_tries:
383            runner.failed = True
384            logging.warning('%s has failed %s times; not restarting',
385                            logger, state.restart_count)
386            return
387
388        # If here, we're going to try restarting.
389        logging.info('%s - restarting', logger)
390        state.last_started = time.time()
391        runner.start()

394    def _handle_logger_death(self, runner, logger):
395        """Called - from the dead runner's stderr relay thread - when a
396        logger process's stderr pipe hits EOF, i.e. the process has exited
397        and its last words have been delivered. Restart it (or declare it
398        failed) right away rather than waiting for the next polling check.
399        """
400        if self.quit_flag or runner.quit_flag:
401            return  # deliberate shutdown, not a death
402
403        # Acquire the lock with a timeout: a manager thread holding it may
404        # be in runner.quit(), joining the very relay thread we're running
405        # in. Re-checking the quit flags lets us bail out instead of
406        # stalling that join.
407        while not self.config_lock.acquire(timeout=0.5):
408            if self.quit_flag or runner.quit_flag:
409                return
410        try:
411            state = self.logger_states.get(logger)
412            if not state or state.runner is not runner:
413                return  # stale notification; logger was reconfigured
414            self._consider_restart(logger, state)
415        finally:
416            self.config_lock.release()

419    def get_status(self):
420        """Return a dict of the current config name and current run status of
421        each logger in the form, e.g.:
422
423        {'s330': {'config':'s330->net', 'status':'RUNNING'},
424         'gyr1': {'config':'gyr1->file', 'status':'FAILED'},
425        }
426
427        Possible status are EXITED, RUNNING, FAILED and STARTING. EXITED
428        is the status when a logger is not 'runnable' - e.g. the 'off'
429        config. STARTING is used when a runner is runnable but is not
430        running and is not FAILED - i.e. we haven't given up on it.
431        """
432        logger_status = {}
433        with self.config_lock:
434            for logger, state in self.logger_states.items():
435                config_name = state.config.get('name', 'no name')
436
437                if not state.runner.is_runnable():
438                    status = 'EXITED'
439                elif state.runner.is_alive():
440                    status = 'RUNNING'
441                elif state.runner.is_failed():
442                    status = 'FAILED'
443                else:
444                    status = 'STARTING'
445                logger_status[logger] = {'config': config_name, 'status': status}
446        return logger_status

419    def get_status(self):
420        """Return a dict of the current config name and current run status of
421        each logger in the form, e.g.:
422
423        {'s330': {'config':'s330->net', 'status':'RUNNING'},
424         'gyr1': {'config':'gyr1->file', 'status':'FAILED'},

426
427        Possible status are EXITED, RUNNING, FAILED and STARTING. EXITED

430        running and is not FAILED - i.e. we haven't given up on it.

436

439                elif state.runner.is_alive():
440                    status = 'RUNNING'
441                elif state.runner.is_failed():
442                    status = 'FAILED'
443                else:
444                    status = 'STARTING'
445                logger_status[logger] = {'config': config_name, 'status': status}
